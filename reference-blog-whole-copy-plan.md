# 참고 블로그 전체 참조 방식 개선 계획

## Context

사용자가 **인터페이스 추가 없이** 참고 블로그 전체를 글 생성 시 참고하는 방식을 원함.

현재 문제:
- 생성된 글이 참고 블로그보다 너무 김
- 문체가 다름
- 구조가 다름

---

## 해결 방법: 프롬프트 강화

인터페이스/타입 추가 없이 **writer-agent.ts 프롬프트만 수정**하여 참고 블로그 전체를 효과적으로 참고하도록 함.

---

## 수정 파일

### `lib/agent/nodes/writer-agent.ts`

**시스템 프롬프트 수정** - 참고 블로그 모방 강조:

```typescript
const systemPrompt = `당신은 블로그 글 작성 전문가입니다.

**중요: 참고 블로그를 완전히 모방하세요**

작성 시 다음을 따르세요:
1. 참고 블로그의 **전체 구조**를 그대로 따르세요
2. 참고 블로그의 **섹션 순서**를 지키세요
3. 참고 블로그의 **문장 길이**와 **문단 길이**를 복제하세요
4. 참고 블로그의 **헤더 사용** 패턴을 따르세요
5. 참고 블로그의 **글 전체 길이**와 비슷하게 작성하세요 (너무 길지 않게!)

스타일 반영:
- 말투(tone), 문장 끝 패턴(endingPattern), 이모지(emojiLevel)를 따르세요
`;
```

**유저 프롬프트 수정** - 참고 블로그 분석 지시:

```typescript
const userPrompt = `다음 정보를 바탕으로 블로그 글을 작성해주세요:

## 참고 블로그 전체 (구조, 길이, 문체를 그대로 모방하세요)
${referencePosts}

---

## 사진 분석 결과 (새 글의 내용)
- 장소: ${places}
- 활동: ${activities}
- 음식: ${foods}
...

---

## 작성 지시
1. 위 **참고 블로그의 구조를 그대로 따르세요**
   - 섹션 개수, 섹션 순서, 섹션 유형을 동일하게
   - 헤더가 있으면 헤더 사용, 없으면 없이
   - 도입부-본문-마무리 구조를 복제

2. **글 전체 길이를 참고 블로그와 비슷하게 맞추세요**
   - 참고 블로그가 전체 몇 문장인지 세어보고 비슷하게
   - 각 섹션의 문장 수도 참고 블로그와 비슷하게
   - 너무 길게 쓰지 마세요!

3. **문장 길이와 문단 길이를 복제하세요**
   - 참고 블로그의 평균 문장 길이를 보고 따르세요
   - 문단이 길면 길게, 짧으면 짧게

4. **사진 분석 결과를 참고 블로그 구조에 맞춰 넣으세요**
   - 내용만 새로운 사진 정보로, 구조/형식은 참고 블로그 그대로
`;
```

---

## 핵심 변경 사항

1. **참고 블로그를 프롬프트 맨 위에 배치** - 모델이 먼저 보도록
2. **구조 복제 지시** - 섹션 순서, 개수, 유형 동일하게
3. **길이 제약** - "너무 길지 않게", "비슷하게" 강조
4. **문장/문단 길이 복제** - 참고 블로그 실제 길이 관찰 지시

---

## 검증

1. `npm run build` - 타입 에러 없음
2. 참고 블로그 입력 후 생성된 글 길이 비교
3. 구조 유사성 확인

---

## 수정 범위

**단일 파일만 수정**: `lib/agent/nodes/writer-agent.ts`

- 시스템 프롬프트: 5-10줄 수정
- 유저 프롬프트: 15-20줄 수정

**타입/인터페이스 추가 없음** - 기존 `referencePosts` 활용
